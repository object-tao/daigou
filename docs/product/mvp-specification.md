# DropPilot MVP 開發規格

## MVP 原則

- 首期先做人工可運營閉環，不接 Mercari、Yahoo、支付網關或物流公司 API。
- 所有涉及金錢、扣款、退款、會員認證、無主包裹和爭議處理的動作都保留人工審核與操作日誌。
- H5 兼容手機與桌面瀏覽器；不做微信小程序。
- APP 先不啟動原生開發，待 H5 閉環穩定後評估 PWA、Flutter 或 React Native。

## 會員端

| 模組 | MVP 功能 | 後續 |
| --- | --- | --- |
| 註冊/會員 | 電郵登入、繁體中文默認、五級會員、會員標籤 | KYC 圖片加密存儲、更多市場語言策略 |
| 代購購物車 | 跨平台商品合併、商品 URL、備註、日本本地運費待補 | 平台 API、自動抓商品資料 |
| Yahoo 代拍 | 會員提交 lot URL、最高出價、授權扣款上限 | 半自動出價與結果同步 |
| 團購 | 會員推薦、線下供應商固定貨同價 | 自動成團規則與開團者獎勵 |
| 集運 | 包裹列表、合箱、一件直發提醒、免倉期展示 | 自助改箱、智能運費估算 |
| 通知 | 郵件為主，WhatsApp 付費加選 | App push、短信 |
| 售後 | 退款/取消/爭議申請，人工審核 | SLA 與自動節點提醒 |

## 後台

| 角色 | 主要職責 |
| --- | --- |
| 超級管理員 | 全部配置、權限、字典、翻譯、審計 |
| 客服 | 會員、報價、售後、消息通知 |
| 倉庫 | 入庫、合箱、一件直發、增值服務、出庫 |
| 財務 | 銀行轉帳審核、匯率、餘額、佣金、退款 |
| 營運 | 團購、優惠券、會員等級、內容與報表 |

## 自動化邊界

- 訂單刷新：MVP 使用服務端定時輪詢與後台列表自動刷新；業務量上來後再評估 WebSocket/SSE。
- 自動扣款：只在會員授權上限內執行，超過條件轉人工確認。
- 無主包裹：進入「公海」狀態，客服/倉庫人工匹配認領。
- 匯率：後台人工設置生效時間，不做實時匯率。

## 已補充的關鍵決策

- 代購與集運狀態以 `order_status_definitions` 為準。
- 包裹查找同時支持日本物流單號、倉庫入庫號、客戶包裹號、合箱批次號、國際物流號。
- 日本本地運費支持預收、人工補收、餘額欠款、下次付款合收。
- 合箱策略、物流渠道、扣款規則、SEO 字段均進入後台可配置模型。
- 客服工單、倉庫掃碼、財務流水納入 MVP 基礎資料結構。

## 合規與安全

- 身份證、付款憑證、包裹圖片應存放在 R2，資料庫只存 URL/Key。
- 敏感圖片與身份資料需要私有桶、短期簽名 URL、訪問日誌。
- 面向香港用戶需要按 PDPO 做資料用途告知、刪除/查閱流程；日本倉儲資料按日本個人資料保護要求保留最小必要資料。
