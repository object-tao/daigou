# DropPilot 完整需求存档

更新时间：2026-06-25  
文档用途：归档 DropPilot 日本代购、集运、会员、物流、分销、财务与管理后台的完整需求。本文不按开发期数拆分，所有已确认、计划支持、暂缓实现和待细化事项均集中记录。

## 1. 产品定位

DropPilot 是面向海外香港用户的日本代购与集运平台，提供从商品提交、人工代购、Yahoo 代拍、日本仓入库、增值服务、合箱打包、自建物流、国际发货、余额支付、积分佣金到售后工单的完整业务系统。

目标用户：

- 香港、澳门、台湾、英国、加拿大、澳大利亚的香港人。
- 以使用繁体中文、港币结算的香港用户为主要目标群体。
- 不面向中国大陆用户。

核心业务：

- 日本商品人工代购。
- Yahoo 人工代拍。
- 日本仓储与集运。
- 免费合箱与纸箱收费。
- 自建国际物流线路。
- 会员余额、积分、佣金。
- 银行转账充值与后续多支付方式。
- 多角色管理后台。
- 多语言 H5 网站。
- SEO 与内容管理。

品牌与域名：

- 品牌：DropPilot。
- 主域名：`droppilot.net`。
- 域名与部署使用 Cloudflare。
- 代码托管在 GitHub：`https://github.com/object-tao/daigou.git`。

## 2. 系统形态

系统分为消费者端和管理后台。

消费者端：

- H5 自适应网站。
- 支持手机和桌面浏览器。
- 提供首页、商品搜索、代购提交、会员中心、订单、包裹、物流、财务、积分、佣金、工单等功能。

管理后台：

- 提供报价、采购、入库、合箱、物流、财务、退款、售后、权限、员工、内容、SEO、基础规则、导出等功能。
- 每个功能独立成菜单，避免多个核心功能混在同一个页面。

暂不需要：

- 微信小程序。
- 中国大陆用户端。

需要后续评估：

- App 形态，包括 PWA、Flutter、React Native 或原生 App。

## 3. 技术与部署

- 前端：React + Vite。
- 后端：Cloudflare Worker。
- 数据库：Cloudflare D1。

部署要求：

## 6. 代购业务

代购模式：

- 以人工代购为主。
- 用户提交商品 URL、平台、商品名称、规格型号、数量、备注。
- 客服或采购人员后台报价、确认、采购。

支持平台：

- Mercari。
- Rakuten Rakuma。
- Yahoo 相关平台。
- Suruga-ya。
- Amazon Japan。
- Animate。
- ZOZOTOWN。
- 其他日本电商平台。

平台对接：

- 当前以人工录入为主。
- 日本电商 API 后续对接。
- Mercari 等平台 API 暂缓自动接入。

购物车：

- 支持跨平台商品加入购物车。
- 跨平台商品可以合并提交。
- 生成订单时可按平台拆分处理。
- 备注需要保留在商品或订单中。
- 若产生日本本地物流费，支持采购后补收。

代购费用：

- 支持按商品金额百分比收服务费。
- 支持每单固定服务费。
- 后台可配置服务费规则。

付款：

- 全额付款后采购。
- 会员余额不允许负数。
- 余额不足时通知会员充值。

取消：

- 未采购可取消。
- 已采购不可取消。
- 取消退款退回余额。

日本本地运费：

- 采购后补收。
- 支持人工补收。
- 支持加入余额欠款。
- 支持下次付款一起收。

议价：

- 个别网店可以议价。
- 议价作为增值服务。
- 议价服务另收增值服务费。

代购订单状态：

| 顺序 | 状态 |
| --- | --- |
| 1 | 待付款 |
| 2 | 已付款 |
| 3 | 待采购 |

| 7 | 待上架 |
| 8 | 已上架 |

| 17 | 已退款 |

## 7. Yahoo 人工代拍

代拍平台：

- 主要支持 Yahoo 拍卖。

代拍流程：

- 用户提交拍卖 URL。
- 用户填写拍品名称。
- 用户设置最高出价。
- 用户设置授权扣款上限。
- 后台人工设置竞拍。
- 后台人工录入竞拍结果。

代拍费用：

- 支持按落札价百分比。
- 支持每单固定费用。
- 后台可配置。

结果处理：

- 得标后生成采购/入库预报。
- 失败或流标后退回余额。
- 代拍失败不扣最终商品款。

## 8. 团购与推荐

团购模式：

- 会员推荐。
- 平台线下与供应商谈好货和价格。
- 支持开团、参团、成团、奖励等业务扩展。

需要支持：

- 团购商品管理。
- 团购价格。
- 参团人数。
- 开团者奖励。
- 会员推荐佣金。

待细化：

- 开团者免费获得商品的具体规则。
- 团购失败退款规则。
- 团购与优惠券是否可叠加。

## 9. 仓库与包裹

仓库：

- 船桥仓为最终出发集运仓。
- 其他地点可作为门市或收货点。
- 仓库、门市需要支持扩展。

包裹编号：

每个包裹支持：

- 日本物流单号。
- 仓库入库号。
- 客户包裹号。
- 合箱批次号。
- 国际物流号。

入库：

- 采购后自动生成入库预报。
- 仓库入库时根据包裹信息自动匹配会员。
- 匹配不到会员时进入无主包裹池/公海。

无主包裹：

- 放入公海。
- 超过 60 天销毁。
- 支持人工认领。

一件直发：

- 用户只有一件货时，不需要上架等待。
- 系统提醒直接转入出库打包区等待发运。

免仓期：

- 默认 30 天。
- 后台可设置。

超仓费：

- 按 1 个代收包裹计算。
- 每天收多少钱由后台设置。

仓库扫码：

- 入库扫码。
- 上架扫码。
- 合箱扫码。
- 打包扫码。
- 出库扫码。
- 手机端可完成扫码操作。

## 10. 合箱

合箱方式：

- 用户自选合箱。
- 客服代选合箱。
- 两者都支持。

合箱规则：

- 支持跨平台包裹合箱。
- 免费合箱。
- 收纸箱费。
- 可选择保留原箱。
- 可选择去除鞋盒。
- 可选择抽真空。
- 可选择加强包装。

纸箱费：

- 按箱型固定价格。
- 后台可配置。

限制：

- 合箱后不允许拆箱重组。

## 11. 增值服务

增值服务包括：

- 拍照。
- 加强包装。
- 抽真空。
- 去鞋盒。
- 保留原箱。
- 议价服务。

费用规则：

- 每项服务分别固定价。

处理流程：

- 用户提交增值服务申请。
- 仓库或客服后台处理。
- 费用进入财务流水。

## 12. 自建物流

物流模式：

- 使用自建物流线路。
- 暂不依赖外部物流公司 API。
- 轨迹由后台录入或内部系统生成。

物流渠道：

- 香港空运。
- 香港海运。
- 英国空运。
- 英国海运。
- 加拿大。
- 澳大利亚。
- 澳门。
- 台湾。

线路配置：

- 计费方式。
- 禁运品。
- 时效。
- 重量限制。
- 体积重公式。

计费：

- 实重和体积重两者取高。
- 运费报价可由后台计算或录入。

物流节点：

- 已打包。
- 已出库。
- 已上飞机/船。
- 到港。
- 清关。
- 派送。
- 签收。

国际物流页面：

- 展示物流报价。
- 展示条件限制。
- 展示禁运品。
- 展示线路说明。
- 支持物流轨迹追踪。

## 25. 数据安全与合规

用户数据：

- 用户资料需要安全存储。
- 身份证号、图片、付款凭证等敏感资料需要加密或受控存储。
- 敏感文件建议存放在 Cloudflare R2，数据库只保存 URL 或 Key。
- 私有文件使用短期签名 URL。
- 访问敏感资料需要记录日志。

合规：

- 香港用户需要考虑 PDPO。
- 日本相关仓储资料需要考虑日本个人资料保护要求。
- 遵循最小必要资料原则。

## 32. 验收原则

系统验收时应确认：

- 消费者端和管理后台清晰分离。
- 每个功能都有独立菜单。
- 核心功能不是单纯展示页面，而是具备数据库读写闭环。
- 会员可以完成注册、登录、代购、代拍、充值、查看订单、查看包裹、查看运单、提交工单。
- 后台可以完成报价、采购、入库、合箱、物流节点、财务审核、退款审核、售后处理。
- 所有资金变化有流水。
- 所有关键后台操作有日志。
- 权限按角色控制。
- 多语言内容可管理。
- SEO 基础字段可管理。
- GitHub Actions 可以自动完成 CI/CD。
